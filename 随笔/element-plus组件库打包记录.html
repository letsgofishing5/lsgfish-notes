node基础知识 ​

node中的path模块与url模块，path模块关注本地文件系统路径的操作，而url模块关注网络资源的URL处理。

path模块 ​

resolve ​

支持多个路径拼接

js
import { resolve } from "path"
let path = resolve("e:/文件路径/文件1.txt", "文件.txt")
console.log('path;', path);//path; e:\文件路径\文件1.txt\文件.txt
path = resolve("e:/文件路径/文件1.txt/", "文件.txt")
console.log('path;', path);//path; e:\文件路径\文件1.txt\文件.txt
path = resolve("e:/文件路径/文件1.txt", "./路径文件.txt")
console.log('path;', path);//path; e:\文件路径\文件1.txt\路径文件.txt
path = resolve("e:/文件路径/文件1.txt/", "./路径文件.txt")
console.log('path;', path);//path; e:\文件路径\文件1.txt\路径文件.txt
path = resolve("e:/文件路径/文件1.txt", "../文件.txt")
console.log('path;', path);//path; e:\文件路径\文件.txt
path = resolve("e:/文件路径/文件1.txt", "文件.txt", "多路径文件.txt")
console.log('path;', path);//path; e:\文件路径\文件1.txt\文件.txt\多路径文件.txt

fileURLToPath ​

js
import { fileURLToPath, URL } from "url"

console.log('import.meta.url:', import.meta.url);//当前文件路径：file:///E:/workspace/test/element-plus-test/node-01/path.mjs

const path = fileURLToPath(import.meta.url)//将文件路径进行转换
console.log('path:', path);//E:\workspace\test\element-plus-test\node-01\path.mjs

dirname ​

返回路径下的目录

js
import { dirname } from "path"

console.log('import.meta.url:', import.meta.url);//当前文件路径：file:///E:/workspace/test/element-plus-test/node-01/path.mjs

const ml = dirname(import.meta.url)
console.log('ml:', ml);//file:///E:/workspace/test/element-plus-test/node-01

gulp 使用 ​

官方文档

基于node，进行自动化构建

入门使用 ​

  1. 初始化项目

    bash
    pnpm init #初始化

pnpm install gulp -D #安装

  2. 新建配置文件

    项目根目录下创建 gulpfile.js 文件，然后输入以下内容

    js
    function defaultTask(cb) {
    console.log('cd进来了');
    // place code for your default task here
    cb();
}

export const build = (cb) => {
    console.log('build');
    cb()
}
export default defaultTask

    然后在package.json中写入以下脚本

    json
    {
  "scripts": {
    "gulp": "gulp",
    "gulp:build": "gulp build"
  },
}

  3. 执行脚本名

    bash
    pnpm gulp #打印信息：cd进来了

pnpm gulp:build #打印信息：build

    gulp 命令对应的是 gulpfile.js 文件默认导出的函数，

    gulp:build 命令对应的是 gulpfile.js 文件中 export const build =()=>{} 方法

串行任务 ​

js
import { series } from "gulp"
const task1 = (cb) => {
    console.log('执行了任务 task1');
    cb()
}
const task2 = (cb) => {
    console.log('执行了任务 task2');
    cb()
}
export default series(task1, task2)
//打印信息
//执行了任务 task1
//执行了任务 task2

任务一个接着一个按顺序执行

并行任务 ​

js
import { series, parallel } from "gulp"
const task1 = (cb) => {
    for (let i = 0; i < 1000; i++) {
        console.log('执行了任务 task1');
        cb()
    }
}
const task2 = (cb) => {
    for (let i = 0; i < 1000; i++) {
        console.log('执行了任务 task2-------');
        cb()
    }
}
export default parallel(task1, task2)

异步执行 ​

在前面的案例代码中，最常见的就是 cb 参数，如果任务（task）不返回任何内容，则必须使用 callback 来指示任务已完成。在如下示例中，callback 将作为唯一一个名为 cb() 的参数传递给你的任务（task）。

js
function callbackTask(cb) {
  // `cb()` should be called by some async work
  cb();
}

exports.default = callbackTask;

还可以返回其他返回值streams、promises、event emitters、child processes, 或 observables。具体案例请前往这里观看

与rollup整合 ​

import { series } from "gulp"
import { rollup, defineConfig } from "rollup"
const rollupTask = async () => {
    const bundle = await rollup({
        input: "./src/index.js",
    })
    bundle.write({
        format: 'cjs', // 配置输出格式
        dir: "./dist/bundle.js", // 配置输出目录
        preserveModules: true, // 该选项将使用原始模块名作为文件名，为所有模块创建单独的 chunk
        // preserveModulesRoot: uiRoot,
        entryFileNames: `[name].mjs` // [name]：入口文件的文件名（不包含扩展名），也就是生产 .mjs 结尾的文件
    })
}
export default series(rollupTask)

执行 gulp 命令后，将执行rollup打包流程

rollup ​

官方定义插件开发

配置文件形式 ​

入门 ​

  1. 创建一个测试项目，项目名称：test-rollup

    bash
    pnpm init #初始化项目

pnpm add rollup -D #安装rollup包

  2. package.json修改

    json
    {
  "name": "test-rollup",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "type": "module",//要改成module
  "scripts": {
   "start": "rollup -c"//添加脚本命令
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "rollup": "^4.17.2"
  }
}

  3. 根目录下创建一个打包入口文件：./src/index.js

  4. 创建需要被打包的 js 文件，然后到处内部数据，在src/index.js 文件中进行引入使用

  5. 根目录下创建 rollup.config.js 配置文件

    js
    export default {
    input:"./src/index.js",
    output:{
        file: 'bundle.js',
		format: 'cjs'
    }
}

  6. 终端中输入脚本命令

    bash
    pnpm start # 运行rollup 打包命令

    然后在根目录下生成了一个名为 bundle.js的文件，该文件就是打包后的js文件

项目环境问题 ​

  1. 添加typescript环境

    1. 添加typescript

      sh
      pnpm add typescript -D

    2. 创建tsconfig.json配置文件

      sh
      npx tsc --init

    3. 配置文件修改后缀为 ts

      1. 添加rollup的ts插件

        sh
        pnpm add @rollup/plugin-typescript tslib

      2. 打包的脚本命令修改为下面的

        sh
        rollup -c rollup.config.ts --configPlugin typescript

      3. 将生成的tsconfig.json中的 module 修改为 nodenext

      4. 将 rollup.config.ts 配置文件添加到 tsconfig.json中的 include

        json
        "include": ["rollup.config.ts"]

  2. 路径别名

    1. 添加插件

      sh
      pnpm install @rollup/plugin-alias -D

    2. 使用方式，观看官方文档示例

      javascript
      import alias from "@rollup/plugin-alias"
export default ({
    plugins: [
          alias({
            entries: [
                { find: '@', replacement: './src' },
            ]
        }),
    ]
})

  3. 自动解析文件后缀

    1. 安装插件

      sh
      pnpm add @rollup/plugin-node-resolve -D

    2. 使用案例看官方文档

      javascript
      import resolve from '@rollup/plugin-node-resolve';
export default ({
    plugins: [
        resolve({ // 解析第三方模块
            extensions: ['.js', '.ts', '.scss', '.css', ".vue", ".ppts"], // 无后缀名引用时，需要识别 .vue 文件
        }),
    ]
})

  4. import导入各类文件时的文件类型提示

    1. 新建global.d.ts文件，在文件中声明文件类型

      typescript
      // globals.d.ts
declare module '*.png' {
    const content: string;
    export default content;
}

declare module '*.jpg' {
    const content: string;
    export default content;
}

  5. 将图片打包成base64

    1. 使用插件@rollup/plugin-image

      sh
      pnpm add @rollup/plugin-image -D

    2. 使用案列，查看官方文档

      javascript
      import image from '@rollup/plugin-image';
export default ({
    plugins: [
        image()
    ]
})

  6. 编译vue文件

    1. 安装插件

      sh
      pnpm install rollup @vitejs/plugin-vue @rollup/plugin-node-resolve rollup-plugin-esbuild -D

      我们知道 Vue 项目打包时候主要做的工作就是将 .vue 文件编译成 JavaScript 文件，如果存在 TypeScript，则也需要将 TypeScript 转换成 JavaScript。我们知道要做什么之后，我们接下来要做的就是去查找和了解相关资料，在 rollup 中是如何做到这些的。在 webpack 中是通过 vue-loader 来识别 .vue 文件的，在 rollup 中则是通过插件来识别的，通常是 rollup-plugin-vue 插件，但因为 rollup-plugin-vue 已经不再维护，推荐使用 @vitejs/plugin-vue；此外 rollup 默认是不能解析 npm 包的，需要通过 @rollup/plugin-node-resolve 插件；TypeScript 代码则是通过 rollup-plugin-esbuild 来进行编译成 JavaScript 的，此插件是一个基于 Esbuild 的插件，速度极快。

      作者：Cobyte

链接：https://juejin.cn/post/7313380227160522763 来源：稀土掘金 著作权归作者所有。商业转载请联系作者获得授权，非商业转载请注明出处。

注意，如果单独将打包内容单独抽取成项目，一定要配置tsconfig.json配置文件，来告诉typescript怎么编译，然后vue源码中的css内容最好抽离出来，因为打包style部分的代码会出问题
